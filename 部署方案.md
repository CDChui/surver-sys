部署方案（surver-sys，Ubuntu 22.04+）

1. 总览
1.1 架构
- 前端：Vite 构建产物 `qianduan/dist`，由 Nginx 提供静态服务
- 后端：Spring Boot Jar（`houduan-0.0.1-SNAPSHOT.jar`）
- 数据库：MySQL 8.x
- 缓存：Redis 6.x/7.x
- 反向代理：Nginx（/ 走前端，/api 走后端）

1.2 端口规划
- 80/443：Nginx
- 8080：后端（建议仅内网或本机）
- 3306：MySQL
- 6379：Redis

2. 服务器环境准备（Ubuntu 22.04+）
2.1 安装基础依赖
```
sudo apt update
sudo apt install -y openjdk-17-jre-headless nginx redis-server mysql-server unzip curl
```

2.2 Node.js（两种方式）
方式 A：不在服务器构建前端
- 在本地/CI 构建前端并上传 `dist` 到服务器

方式 B：在服务器构建
```
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

2.3 创建运行用户与目录
```
sudo useradd -r -s /bin/false surver
sudo mkdir -p /opt/surver-sys
sudo chown -R $USER:$USER /opt/surver-sys
```
将代码部署到 `/opt/surver-sys`：
```
rsync -av ./ /opt/surver-sys/
```

3. 数据库与 Redis
3.1 MySQL 初始化
```
sudo mysql -e "CREATE DATABASE questionnaire DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
sudo mysql -e "CREATE USER 'survey'@'%' IDENTIFIED BY 'strong_password';"
sudo mysql -e "GRANT ALL PRIVILEGES ON questionnaire.* TO 'survey'@'%';"
sudo mysql -e "FLUSH PRIVILEGES;"
```

3.2 Redis 基础配置（可选）
- 默认本机访问即可
- 如需远程访问，配置 `bind` 与 `requirepass`

4. 后端部署
4.1 构建
```
cd /opt/surver-sys/houduan
chmod +x mvnw
./mvnw -DskipTests package
```
产物：`/opt/surver-sys/houduan/target/houduan-0.0.1-SNAPSHOT.jar`

4.2 环境变量文件
创建 `/etc/surver-sys/backend.env`：
```
SERVER_PORT=8080
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=questionnaire
DB_USER=survey
DB_PASSWORD=strong_password
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0
APP_JWT_SECRET=change-this-to-a-strong-secret
APP_ADMIN_INIT_USERNAME=admin
APP_ADMIN_INIT_PASSWORD=change_me
APP_SSO_MOCK_ENABLED=false
```
```
sudo mkdir -p /etc/surver-sys
sudo chown root:root /etc/surver-sys
sudo chmod 700 /etc/surver-sys
sudo chmod 600 /etc/surver-sys/backend.env
```

4.3 systemd 服务
创建 `/etc/systemd/system/surver-sys-backend.service`：
```
[Unit]
Description=Surver Sys Backend
After=network.target mysql.service redis.service

[Service]
Type=simple
User=surver
WorkingDirectory=/opt/surver-sys/houduan
EnvironmentFile=/etc/surver-sys/backend.env
ExecStart=/usr/bin/java -jar /opt/surver-sys/houduan/target/houduan-0.0.1-SNAPSHOT.jar
Restart=always
RestartSec=5
SuccessExitStatus=143

[Install]
WantedBy=multi-user.target
```
启用并启动：
```
sudo systemctl daemon-reload
sudo systemctl enable surver-sys-backend
sudo systemctl start surver-sys-backend
sudo systemctl status surver-sys-backend
```

4.4 初始化说明
- Flyway 会自动创建表（`houduan/src/main/resources/db/migration`）
- 首次登录后请修改管理员密码

5. 前端部署
5.1 构建（在服务器构建时）
```
cd /opt/surver-sys/qianduan
npm install
```
创建 `.env.production`：
```
VITE_USE_REAL_API=true
VITE_API_BASE=/api
```
构建：
```
npm run build
```

5.2 Nginx 配置
创建 `/etc/nginx/conf.d/surver-sys.conf`：
```
server {
  listen 80;
  server_name your.domain.com;

  root /opt/surver-sys/qianduan/dist;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:8080/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```
检查并重载：
```
sudo nginx -t
sudo systemctl reload nginx
```

5.3 HTTPS（可选）
```
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your.domain.com
```

6. 系统设置与标题 LOGO
- 标题文字：读取“系统设置 -> 基础设置 -> 系统名称”
- 标题 LOGO：上传“网页标题 LOGO”
- 若移动端未立即显示，建议强制刷新或清缓存

7. 验证清单
- 前端首页可访问，路由刷新不 404
- `curl http://127.0.0.1:8080/api/settings/public` 返回 200
- 管理后台登录成功
- 系统设置保存后重启仍生效
- 问卷创建、答题、日志、统计功能正常

8. 备份与恢复
8.1 MySQL 备份
```
mysqldump -u survey -p questionnaire > questionnaire_backup.sql
```
8.2 恢复
```
mysql -u survey -p questionnaire < questionnaire_backup.sql
```

9. 常见问题
- 后端启动失败：检查 `APP_JWT_SECRET`、数据库/Redis 连接
- 前端接口 404：确认 Nginx `/api` 反代配置
- 图标不更新：浏览器缓存，建议强刷或清缓存

10. 上线流程建议
- 先在测试环境部署并验证
- 备份数据库
- 上线窗口停止旧服务，发布新包
- 校验登录、问卷流程、日志与统计
