## 4. 启动与联调

### 4.1 前端启动

在 `qianduan` 目录执行：

```powershell
npm run dev
```

常用命令：

```powershell
npm run build-only
npm run type-check
```

### 4.2 后端启动

在 `houduan` 目录执行（推荐）：

```powershell
.\run.ps1 -NoDeps -SkipTests spring-boot:run
```

仅编译验证：

```powershell
.\mvnw.cmd -q -DskipTests compile

VITE_PROXY_TARGET=http://127.0.0.1:8080
```

## 5. 功能分类说明

### 5.1 登录认证与会话

本地登录：
- 页面：`/local-admin/login`
- 接口：`POST /api/auth/local/login`

第三方登录（OAuth/IAM）：
- 前端根据系统设置中的 `authIntegration` 生成授权地址并发起跳转。
- 回调页：`/auth/sso/callback`。
- OAuth `state` 中保存 `redirectPath`，用于回到原访问页。

未登录自动跳转策略：
- 访问 `meta.requiresAuth=true`（后台）或 `meta.requiresLogin=true`（移动端）页面时：
1. 优先请求 `GET /api/settings/public-auth`。
2. 若第三方可用，直接发起第三方登录。
3. 否则回退本地登录页。

退出登录：
- 接口：`POST /api/auth/logout`
- 退出页：`/auth/logout`
- 管理员与普通用户展示不同按钮文案与跳转目标。
- 使用 `logout-context` 作为 query 丢失时的回退机制。

相关文件：
- `src/router/index.ts`
- `src/utils/oauth-login.ts`
- `src/views/auth/SsoCallback.vue`
- `src/views/auth/Logout.vue`
- `src/views/admin/Login.vue`
- `src/api/request.ts`
- `src/api/settings.ts`

### 5.2 用户管理

用户类型：
- 本地用户（可本地密码管理）
- 第三方用户（账号/名称由第三方提供，不可改）

用户字段：
- 账号（原“用户名”）
- 名称（原“姓名”）
- 角色
- 状态
- 备注（新增）
- localAccount（是否本地账号）

功能：

- 管理员可重置本地用户密码。
- 本地用户可在“修改密码”页修改自己的密码。
- 用户页左侧按“本地用户 / 第三方用户”分类筛选。

相关前端文件：

- `src/views/admin/ChangePassword.vue`

- `src/stores/user.ts`

- 个人首页展示：账号、名称、已答问卷记录。

## 8. 常见问题排查

### 8.1 登录页反复跳转/刷新

现象：登录页不断刷新。

原因：`/settings/public-auth` 的 401 被全局拦截器误判为需要登录重定向。

修复要点：
- `getPublicAuthSettings()` 请求加 `X-Skip-Auth-Redirect: 1`
- 拦截器对该请求 401 不触发重定向

相关文件：
- `src/api/settings.ts`
- `src/api/request.ts`

### 8.2 后端未启动导致前端报网络错误

检查顺序：
1. 后端是否监听 `8080`
2. `VITE_PROXY_TARGET` 是否正确
3. 浏览器访问 `http://127.0.0.1:8080/api/test` 是否通
