# 前端维护文档

## 1. 文档信息
- 项目：问卷调查系统前端（`qianduan`）
- 技术栈：Vue 3 + Vite + TypeScript + Pinia + Vue Router + Element Plus + Vant
- 最近更新日期：2026-03-08
- 当前维护重点：后台管理完善、系统设置二级化、通用 OAuth 平台配置（域名直填）

## 2. 关键目录与职责
- `src/api`
  - `request.ts`：Axios 实例、请求/响应拦截、业务码处理
  - `auth.ts`：本地登录 mock 与第三方回调登录接口封装
  - `settings.ts`：系统设置读写（含基础设置与第三方对接）
  - 其余：问卷、用户、日志、问卷授权接口
- `src/stores`
  - `settings.ts`：系统设置状态与本地存储兼容迁移
  - `auth.ts` / `permission.ts`：登录态与权限控制
- `src/router/index.ts`
  - 路由定义与守卫
- `src/utils/auth-redirect.ts`
  - 登录前后跳转路径清洗与回跳规则
- `src/utils/oauth-login.ts`
  - 第三方登录入口构建、state 管理、默认平台解析
- `src/views/auth`
  - `SsoCallback.vue`：SSO 回调处理页（解析 `code/state` 并完成登录）
  - `Logout.vue`：退出结果页（提示“您已退出登录”）
- `src/views/admin/settings`
  - `BasicSettings.vue`：基础设置子页
  - `ThirdPartyIntegration.vue`：第三方对接子页（通用 OAuth + IAM 模板）

## 3. 路由说明（后台）
- `/admin/settings`：系统设置父路由（默认跳转基础设置）
- `/admin/settings/basic`：基础设置
- `/admin/settings/integration`：第三方对接
- `/m/surveys/:id`：问卷答题页（当前要求登录后访问）
- `/auth/logout`：退出结果页

### 3.1 登录回跳规则
- 未登录访问 `/admin/*`：
  - 启用第三方默认登录时直接拉起第三方认证；
  - 未启用时跳转本地登录页并携带 `redirect` 参数。
- 未登录访问 `/m/surveys/:id`：跳转登录页并携带 `redirect` 参数，普通用户登录后回到该问卷页。
- 普通用户（`ROLE1`）登录后若目标是后台地址（`/admin/*`），统一跳转 `/m`。
- 管理员用户（`ROLE2/ROLE3`）访问登录页后，默认回跳后台（无 `redirect` 时到 `/admin/dashboard`）。

### 3.2 后台第三方默认登录策略
- 当 `authIntegration.loginMode !== LOCAL_ONLY` 且默认认证平台可用时：
  - 未登录用户访问后台任意页面（`/admin/*`）会直接发起第三方认证授权（不进入本地登录页）。
- 访问 `/local-admin/login` 时始终使用本地登录流程（不强制跳第三方）。
- 本地登录页提供“使用第三方认证登录”入口（按钮），点击后按默认认证平台发起授权登录。
- 若第三方配置不可用或授权地址异常，回退到本地登录页。

## 4. 系统设置模块

### 4.1 基础设置子页
- 文件：`src/views/admin/settings/BasicSettings.vue`
- 字段：
  - `systemName`
  - `systemDomain`
  - `defaultPageSize`
  - `enableLog`
  - `enableResumeDraft`
  - `allowDuplicateSubmit`

### 4.2 第三方对接子页（通用 OAuth）
- 文件：`src/views/admin/settings/ThirdPartyIntegration.vue`
- 页面结构：
  - 全局认证设置
    - 登录模式：`LOCAL_ONLY` / `LOCAL_AND_OAUTH` / `OAUTH_ONLY`
    - 默认认证平台 `defaultProviderId`
    - 首次登录自动创建用户 `autoCreateUser`
    - 新用户默认角色 `defaultRole`
  - 认证平台列表
    - 支持新增、编辑、复制、删除、单平台校验
  - 平台配置弹窗
    - 协议类型：`IAM_TEMPLATE` / `OAUTH2` / `OIDC`
    - IAM 模板不再选择环境，管理员直接填写认证域名
    - IAM 模板默认唯一标识字段：`employeeNumber`（学工号）
    - IAM 模板新增平台时默认地址：
      - `redirect_uri = {systemDomain}/auth/sso/callback`
      - `logoutRedirectUri = {systemDomain}/auth/logout`
    - 提供接口地址预览

### 4.3 IAM 兼容策略
- `settings` store 保留旧字段 `oauth`，并新增 `authIntegration`。
- 读取旧数据时：
  - 若只有 `oauth`，会自动迁移生成 `authIntegration.providers`。
- 保存新数据时：
  - 会从 `authIntegration` 反向回写 `oauth`（供旧逻辑兼容）。
- 兼容字段说明：
  - `environment` 字段仍在数据结构中保留，用于历史数据兼容；当前前端统一按 `CUSTOM` 处理，不在页面展示环境选择。

### 4.4 第三方登录闭环（当前实现）
- 发起登录：
  - 后台未登录访问触发默认平台 `authorize_url` 跳转。
- 回调处理：
  - 路由 `/auth/sso/callback` 解析 `code/state`，校验并消费 `state`，再调用 `api/auth.ts` 的 `oauthCallbackLogin`。
  - `USE_REAL_API=true` 时调用后端 `/auth/oauth/callback`。
  - `USE_REAL_API=false` 时使用前端 mock 登录结果。
- 登录完成后：
  - 按角色与 `redirect` 规则跳转目标页。

## 5. 设置数据结构（当前）
- 文件：`src/stores/settings.ts`
- `SystemSettings` 核心结构：

```ts
export interface SystemSettings {
  systemName: string
  systemDomain: string
  defaultPageSize: number
  enableLog: boolean
  enableResumeDraft: boolean
  allowDuplicateSubmit: boolean
  oauth: OauthSettings
  authIntegration: AuthIntegrationSettings
}
```

- `AuthIntegrationSettings`：

```ts
export interface AuthIntegrationSettings {
  loginMode: 'LOCAL_ONLY' | 'LOCAL_AND_OAUTH' | 'OAUTH_ONLY'
  defaultProviderId: string
  autoCreateUser: boolean
  defaultRole: 'ROLE1' | 'ROLE2' | 'ROLE3'
  providers: OauthProviderConfig[]
}
```

- `OauthProviderConfig` 关键字段：

```ts
export interface OauthProviderConfig {
  id: string
  name: string
  protocol: 'IAM_TEMPLATE' | 'OAUTH2' | 'OIDC'
  enabled: boolean
  priority: number
  environment: 'TEST' | 'PROD' | 'CUSTOM'
  authDomain: string
  clientId: string
  clientSecret: string
  scope: string
  redirectUri: string
  logoutRedirectUri: string
  authorizeUrl: string
  tokenUrl: string
  userInfoUrl: string
  refreshUrl: string
  revokeUrl: string
  userIdField: string
  realNameField: string
  emailField: string
}
```
