# 问卷系统后端任务拆分 V2（前端已完成对齐版）

## 1. 文档信息

- 基线输入：
  - 附件1《后端任务拆分.docx》（2026-03-05）

- 更新日期：2026-03-08

## 2. 前后端对齐结论（V1 -> V2 调整项）
1. OAuth 回调接口以当前前端为准：`POST /api/auth/oauth/callback`（不是 V1 的 `GET /auth/sso/callback`）。
2. 公开问卷与提交接口以当前前端为准：
   - `GET /api/surveys/{id}/public`
   - `POST /api/surveys/{id}/responses`
3. 问卷列表接口以当前前端为准：`GET /api/surveys`（服务端按角色做可见性过滤）。
4. 系统设置接口固定为：`GET /api/settings`、`POST /api/settings`，并支持 `oauth + authIntegration` 双结构兼容。
5. 用户管理接口按前端已接入能力补齐：`GET/POST/PUT/DELETE /api/users`、`POST /api/users/{id}/role`、`POST /api/users/{id}/status`。
6. 日志接口按前端已接入能力补齐：`GET /api/logs`、`POST /api/logs`。
7. 业务码必须兼容前端拦截器：`20000/40001/40009/40011/40101/40102/40301/40404/50000`。

## 3. 实施范围（按优先级）
- `P0（联调阻塞）`：认证、设置、用户、问卷主流程、问卷授权、公开答题、提交、防重复、错误码、统一响应。
- `P1（管理闭环）`：问卷发布/关闭/删除接口化、统计、导出、日志查询与审计完善。
- `P2（稳定性）`：压测、指标监控、异常告警、性能优化、灰度与回滚策略。

## 5. 数据与并发方案（落库要求）
1. 沿用 V1 核心表：`sys_user`、`survey_info`、`survey_auth`、`survey_answer`、`sys_audit_log`。
2. 新增系统设置表（V2 必需）：
   - `sys_settings`（单行配置）：
   - 基础字段：`system_name/system_domain/default_page_size/enable_log/enable_resume_draft/allow_duplicate_submit`
   - JSON 字段：`oauth_json/auth_integration_json`
3. 防重复提交：
   - DB 唯一键：`uk_survey_user (survey_id, user_id)`。
   - Redis 锁：`survey:submit:lock:{surveyId}:{userId}`，TTL 10s。
4. 软上限：
   - 入场票据：`survey:entry:{surveyId}:{userId}`，TTL 30min。
   - 计数：`survey:quota:count:{surveyId}`。
5. JWT 与注销：
   - 黑名单：`jwt:blacklist:{jti}`，TTL=token 剩余时间。

## 8. 联调验收清单（必须通过）
1. 未登录访问 `/admin/*` 时，后端返回 40101/40102，前端可自动跳转登录入口。
2. OAuth 回调成功后，返回结构必须包含：`token/role/username/realName/userId`。
3. `GET /api/settings` 与 `POST /api/settings` 能正确读写 `authIntegration.providers`。
4. `GET /api/surveys` 对 `ROLE2` 仅返回本人+授权问卷；`ROLE3` 可返回全部。
5. `GET /api/surveys/{id}/public` 在名额满时返回 `40011`。
6. 同一用户重复提交同问卷返回 `40009`。
7. `GET /api/surveys/{id}/export` 返回 `xlsx` 二进制可被前端下载。
