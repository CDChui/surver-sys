- 项目：问卷调查系统后端（`houduan`）

## 3. 接口说明（与前端联调）
- `/api/auth/local/login`：本地登录
- `/api/auth/oauth/callback`：第三方回调换取内部 JWT
- `/api/auth/logout`：注销（token 黑名单）
- `/api/settings`：系统设置读取与保存（含基础设置+第三方对接）
- `/api/users`：用户列表/创建/更新/删除
- `/api/users/{id}/role`：角色变更
- `/api/users/{id}/status`：状态变更
- `/api/surveys`、`/api/surveys/{id}`：问卷创建、列表、详情、编辑
- `/api/surveys/{id}/auth`：问卷授权用户管理
- `/api/surveys/{id}/public`：移动端公开问卷详情
- `/api/surveys/{id}/responses`：移动端提交答卷
- `/api/surveys/{id}/stats`：统计
- `/api/surveys/{id}/export`：导出 Excel
- `/api/logs`：日志查询与记录

### 3.1 鉴权与权限规则
- 角色定义：`ROLE1`（普通用户）、`ROLE2`（业务管理员）、`ROLE3`（系统管理员）。
- JWT 需包含：`sub/uid/loginName/displayName/role/jti/exp`。
- 后端强制执行接口级 + 资源级权限，不依赖前端按钮隐藏。
- 前端关键行为依赖错误码：
  - `40101`：未登录/无效 token
  - `40102`：token 过期或吊销
  - `40301`：无权限
  - `40009`：重复提交
  - `40011`：名额已满

### 3.2 统一响应规范
- 所有 JSON 接口统一：
```json
{
  "code": 20000,
  "message": "success",
  "data": {}
}
```
- 推荐错误码：`40001/40009/40011/40101/40102/40301/40404/50000`。

### 3.3 当前实现状态（2026-03-08）
- 已落库模块：
  - `settings`：读写 `sys_settings`，并保存 `oauth/authIntegration` JSON。
  - `users`：读写 `sys_user`，应用启动后自动初始化本地管理员（若无本地账号）。
  - `surveys`：读写 `survey_info/survey_auth/survey_answer`，支持 CRUD、授权、提交、统计、导出。
  - `logs`：读写 `sys_audit_log` 并关联 `sys_user`。
- 已落地安全行为：
  - JWT 过滤器支持 token 解析与黑名单校验。
  - token 过期/吊销返回业务码（`40102`），缺失 token 由 Security 入口返回（`40101`）。
- 已落地 Redis 协同能力（问卷提交流程）：
  - 入场票据：`survey:entry:{surveyId}:{userId}`（TTL 30 分钟）
  - 提交互斥锁：`survey:submit:lock:{surveyId}:{userId}`（TTL 10 秒）
  - 软上限计数：`survey:quota:count:{surveyId}`
- 已落地审计日志自动写入（`sys_audit_log`）：
  - `SYSTEM`：登录/退出/系统设置更新
  - `USER`：创建/编辑/删除/角色与状态变更
  - `SURVEY`：创建/编辑/发布/关闭/删除/答卷提交/导出
  - `PERMISSION`：问卷授权与撤销
- 当前限制：
  - 前端当前未显式传 `entryToken`，后端默认使用服务端缓存票据校验；若请求体带 `entryToken` 则会进行一致性校验。
  - Redis 不可用时，提交流程会失败（当前未实现降级策略）。

## 4. 系统设置模块

### 4.1 基础设置
- 字段：
  - `systemName`
  - `systemDomain`
  - `defaultPageSize`
  - `enableLog`
  - `enableResumeDraft`
  - `allowDuplicateSubmit`

### 4.2 第三方对接（通用 OAuth）
- 全局认证配置：
  - `loginMode`：`LOCAL_ONLY` / `LOCAL_AND_OAUTH` / `OAUTH_ONLY`
  - `defaultProviderId`
  - `autoCreateUser`
  - `defaultRole`
- 平台配置字段：
  - `id/name/protocol/enabled/priority`
  - `authDomain/clientId/clientSecret/scope`
  - `redirectUri/logoutRedirectUri`
  - `authorizeUrl/tokenUrl/userInfoUrl/refreshUrl/revokeUrl`
  - `userIdField/realNameField/emailField`

### 4.3 IAM 兼容策略
- 后端存储同时保留：
  - 旧结构：`oauth`
  - 新结构：`authIntegration`
- 保存时建议以 `authIntegration` 为主，并回写 `oauth` 兼容旧逻辑。
- `IAM_TEMPLATE` 协议默认 `userIdField=employeeNumber`，历史 `uid` 需兼容迁移。

### 4.4 第三方登录闭环
- 前端路由回调：`/auth/sso/callback`（前端页面）
- 后端回调接口：`POST /api/auth/oauth/callback`
- 闭环流程：校验 `providerId/code/state` -> 换取用户信息 -> 映射系统用户 -> 签发 JWT -> 返回前端跳转。
